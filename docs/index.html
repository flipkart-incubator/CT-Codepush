Quick Start Guide

1

Get Your Access Key

Obtain your CodePush access key from the dashboard or CLI. This will be used for authentication.

$ code-push access-key add "My API Key"
2

Authenticate Your Requests

Include your access key in the Authorization header for all API requests.

Authorization: Bearer YOUR_ACCESS_KEY
3

Try the Interactive Documentation

Use our Swagger UI to explore endpoints and test API calls directly in your browser.

Open Interactive Docs
4

Start Building

Create your first app, set up deployments, and start managing releases.

POST /apps
{
  "name": "MyApp",
  "description": "My awesome mobile app"
}

API Endpoints Overview

Authentication

  • POST /auth/login/google - Google OAuth login
  • GET /auth/accesskey - Get access key
  • GET /account - Get user profile

Apps

  • GET /apps - List applications
  • POST /apps - Create application
  • PATCH /apps/{name} - Update application
  • DELETE /apps/{name} - Delete application

Deployments

  • GET /apps/{name}/deployments - List deployments
  • POST /apps/{name}/deployments - Create deployment
  • DELETE /apps/{name}/deployments/{env} - Delete deployment

Releases

  • POST /.../release - Upload release
  • POST /.../promote/{target} - Promote release
  • POST /.../rollback - Rollback release
  • GET /.../history - Get release history

Collaborators

  • GET /apps/{name}/collaborators - List collaborators
  • POST /.../collaborators/{email} - Add collaborator
  • DELETE /.../collaborators/{email} - Remove collaborator

Metrics

  • GET /.../metrics - Get deployment metrics
  • Track downloads, installs, and failures
  • Monitor active user adoption
